31 /*
32  * The System Control Space (SCS) is a memory-mapped 4KB address space that
33  * provides 32-bit registers for configuration, status reporting and control.
34  * The SCS registers divide into the following groups:
35  * - system control and identification
36  * - the CPUID processor identification space
37  * - system configuration and status
38  * - fault reporting
39  * - a system timer, SysTick
40  * - a Nested Vectored Interrupt Controller (NVIC)
41  * - a Protected Memory System Architecture (PMSA)
42  * - system debug.
43  */

150 /*
151  * System Control Space (SCS) => System timer register support in the SCS.
152  * To configure SysTick, load the interval required between SysTick events to
153  * the SysTick Reload Value register. The timer interrupt, or COUNTFLAG bit in
154  * the SysTick Control and Status register, is activated on the transition from
155  * 1 to 0, therefore it activates every n+1 clock ticks. If you require a
156  * period of 100, write 99 to the SysTick Reload Value register. The SysTick
157  * Reload Value register supports values between 0x1 and 0x00FFFFFF.
158  *
159  * If you want to use SysTick to generate an event at a timed interval, for
160  * example 1ms, you can use the SysTick Calibration Value Register to scale
161  * your value for the Reload register. The SysTick Calibration Value Register
162  * is a read-only register that contains the number of pulses for a period of
163  * 10ms, in the TENMS field, bits[23:0].
164  *
165  * This register also has a SKEW bit. Bit[30] == 1 indicates that the
166  * calibration for 10ms in the TENMS section is not exactly 10ms due to clock
167  * frequency. Bit[31] == 1 indicates that the reference clock is not provided.
168  */
169 /*
170  * SysTick Control and Status Register (CSR).
171  * Purpose Controls the system timer and provides status data.
172  * Usage constraints: There are no usage constraints.
173  * Configurations Always implemented.
174 */
175 #define SCS_SYST_CSR MMIO32(SCS_BASE + 0x10)
