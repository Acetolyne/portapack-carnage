24 /* Notes to self (or others, welcome !):
25  * Sharebrained wrote in matched_filter.hpp that taps should be those of a complex low-pass filter combined with a complex sinusoid, so
26  * that the filter shifts the spectrum where we want (signal of interest around 0Hz).
27  *
28  * In this baseband processor, after decim_0 and decim_1, the signal ends up being sampled at 38400Hz (2457600 / 8 / 8)
29  * Since the applied shift in ui_sonde.cpp is -fs/4 = -2457600/4 = -614400Hz to avoid the DC spike, the FSK signal ends up being
30  * shifted by 614400 / 8 / 8 = 9600Hz. So decim_1_out should look like this:
31  *
32  * _______________|______/'\______
33  * -C A B C
34  *
35  * A is the DC spike at 0Hz
36  * B is the FSK signal shifted right at 9600Hz
37  * C is the bandwidth edge at 19200Hz
38  *
39  * Taps should be computed to shift the whole spectrum by -9600Hz ("left") so that it looks like this:
40  *
41  * ______________/'\______________
42  * -C D C
43  *
44  * Anything unwanted (like A) should have been filtered off
45  * D is B around 0Hz now
46  *
47  * Then the clock_recovery function should be happy :)
48  *
49  * Mathworks.com says:
50  * In the case of a single-rate FIR design, we simply multiply each set of coefficients by (aka 'heterodyne with') a complex exponential.
51  *
52  * Can SciPy's remez function be used for this ? See tools/firtest.py
53  * GnuRadio's firdes only outputs an odd number of taps
54  *
55  * ---------------------------------------------------------------------
56  *
57  * Looking at the AIS baseband processor:
58  *
59  * Copied everything necessary to get decim_1_out (so same 8 * 8 = 64 decimation factor)
60  * The samplerate is also the same (2457600)
61  * After the matching filter, the data is decimated by 2 so the final samplerate for clock_recovery is 38400 / 2 = 19200Hz.
62  * Like here, the shift used is fs/4, so decim_1_out should be looking similar.
63  * The AIS signal deviates by 2400 (4800Hz signal width), the symbol rate is 9600.
64  *
65  * The matched filter's input samplerate is 38400Hz, to get a 9600Hz shift it must use 4 taps ?
66  * To obtain unity gain, the sinusoid length must be / by the number of taps ?
67  *
68  * See ais_baseband.hpp
69  *
70  * */
