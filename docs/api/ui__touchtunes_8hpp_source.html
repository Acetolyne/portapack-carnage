28 // The coding in notpike's script is quite complex, using multiple LUTs to form the data sent to the YSO.
29 // The format is actually very simple if it is rather seen as short and long gaps between pulses (as seen in many OOK remotes).
30 // The frames and data rate suspiciously match the NEC infrared protocol (http://www.sbprojects.com/knowledge/ir/nec.php) without
31 // the address complement. The exact data rate would be 1786 baud (560us/fragment).
32 // NotPike: The data rate should be 1786 but the remote was transmitting at 1766

58 // The sync word seems to be SLSLLLSL (01011101, or 0xBA reversed)
59 // The pin # is sent LSB first
60 // The button data seems to be SLLLLSSSLSSSSLLL (0111100010000111 and a terminating pulse, or 0x7887 reversed)
61 // The hex data only seems scrambled because of the shift induced by the short or long gaps (10 or 1000)
62 // The radio frame's duration depends on the value of the bits
